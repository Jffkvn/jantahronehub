# HR Setup Workspace Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a secure HR Setup workspace where HR and super administrators can manage departments, job titles and pay grades, then assign them to employees.

**Architecture:** Existing canonical setup tables remain the source of truth. A new migration adds atomic, audited management functions and removes direct client mutation privileges. A typed React module exposes setup management under HR, while the employee form consumes the same canonical data and persists pay-grade assignments.

**Tech Stack:** PostgreSQL/Supabase RLS and RPCs, pgTAP, React 19, TypeScript, React Router, TanStack Query, React Hook Form, Zod, Vitest, Testing Library and Playwright.
