# Projects and Daily Tracker

## Purpose

Projects is the operational home for each job. Daily Tracker records day-to-day field progress against those projects; it is not where projects are created or assigned.

## Responsibilities

| Role | Project responsibility |
| --- | --- |
| CFO | Create a project, appoint or replace the primary Project Manager, assign coordinators when needed, and review financial controls. |
| Project Manager | Create projects, manage the status and operational details of assigned projects, add or remove coordinators with a reason, and review coordinator updates. |
| Project Coordinator | View assigned projects, submit daily updates, and request cash or inventory only for projects within their assignment scope. |
| Super administrator | Support and oversee configured workflows without replacing the accountable operational roles. |

One project can have one active primary Project Manager and multiple active coordinators.

## Creating a project

Open **Operations → Projects → Create project**. Complete the three sections:

1. **Project identity** — project code, name, client, site, and reference details.
2. **Schedule and controls** — planned dates, status, health, budget, and operational notes.
3. **Project team** — primary Project Manager, one or more coordinators, and the audit reason.

Project Manager creators are assigned as the primary Project Manager automatically. CFO creators may appoint a Project Manager immediately or add one later.

## Assignment behaviour

When a Project Manager or coordinator is assigned:

- the person receives one in-app Project assignment notification;
- the project appears in their Projects list;
- their real display name and project role appear on the Team tab;
- the assignment and its reason remain in project history;
- coordinators can select the same planned, active, or on-hold project in both Cash and Inventory request forms.

Removing or replacing an assignment requires a reason. Historical assignment records are retained.

## Using the project workspace

- **Summary** shows schedule, team count, cash reconciliation, and inventory/equipment reconciliation.
- **Team** shows the primary Project Manager, coordinators, controls, and assignment history.
- **Daily Updates** holds coordinator submissions and Project Manager review.
- **Cash** and **Inventory & Equipment** reconcile the canonical project-linked ledgers.
- **Documents** stores project contracts, site evidence, and completion records.
- **History** shows the auditable project activity timeline.

## Daily Tracker workflow

The assigned coordinator records the daily field update. The assigned Project Manager reviews it and either endorses it or requests revision with feedback. Project creation and assignment remain in Projects so that operational ownership, cash, inventory, documents, and history stay attached to one canonical project record.
